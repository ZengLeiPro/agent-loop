创建运行

权限模式区别： 这些取值来自 @anthropic-ai/claude-agent-sdk 的 PermissionMode 定义。
  • default：SDK 标准权限行为；危险操作会请求确认。
  • acceptEdits：自动接受文件编辑，仍保留其他权限控制；这是本 UI 默认值。
  • plan：规划模式，不执行实际修改。
  • dontAsk：不会弹出权限请求；未预先允许的操作会被拒绝，所以它不是 YOLO，反而更适合无交互/保守运行。
  • auto：使用模型分类器自动批准或拒绝权限请求。
  • bypassPermissions：跳过全部权限检查；这是最 YOLO 的模式，仅在完全信任运行环境和任务时使用。

API 端点、API Key、effort、thinking tokens 留空时不会提交给 SDK，继续使用环境变量、登录状态或 SDK 默认值。

dry run 只写入本地 .agent-loop/ 状态文件，用来预览 run 配置；不会调用 Claude Agent SDK，也不会消耗 API。要真实启动 Planner → Worker → Judge 时再取消勾选。

启用后只让 Planner 生成 spec / PRD，暂停给人工检查；确认后再继续后续 Worker / Judge loop。

这里的配置会直接提交到运行 API。只有在已经安装依赖并配置好 Claude Agent SDK 凭据后，才建议关闭 dry run。

提示语编辑

通常只需要编辑 Agent 系统提示语；保存后会写入当前仓库的 .agent-loop/prompts/，后续运行立即生效。阶段任务模板属于高级入口，默认折叠。

Agent 系统提示语

系统提示语定义每个角色“是谁、必须遵守哪些规则”：例如 Planner 如何拆 PRD、Worker 如何提交、Judge 如何判定 PASS / FAIL。它会作为该角色的 system prompt 使用。

高级：阶段任务模板（通常不用改）

这些模板不是另一套角色规则，而是 orchestrator 每轮发给 Agent 的短任务消息，用来注入用户任务或轮次编号。默认模板只做最小调度；除非你要改 loop 协议或占位符传递方式，否则保持默认即可。可使用占位符：{{prompt}} 表示用户任务，{{round}} 表示当前轮次。
